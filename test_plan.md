# План тестирования 

## 1. Цели тестирования

### Функциональное тестирование

- Проверить корректность операций CRUD с задачами:
  - добавление;
  - редактирование;
  - удаление;
  - отображение в таблице.
- Проверить работу поиска и фильтрации задач по названию.
- Проверить корректность сохранения и загрузки данных в формате JSON.
- Проверить обновление мини-статистики:
  - общее количество задач;
  - количество выполненных задач;
  - процент выполненных.

### Пользовательское тестирование 

- Оценить понятность интерфейса для новых пользователей.
- Проверить, насколько легко:
  - добавить задачу;
  - найти задачу;
  - понять статистику.
- Собрать предложения по улучшению интерфейса и функционала.

---

## 2. Объект тестирования



Основные модули:

- Вкладка **«Задачи»**:
  - таблица задач;
  - кнопки «Добавить», «Редактировать», «Удалить»;
  - поле поиска.
- Вкладка **«Статистика»**:
  - количество задач;
  - количество выполненных;
  - процент выполненных.
- Окно **добавления/редактирования** задачи.
- Механизм сохранения и загрузки задач в формате JSON.

---

## 3. Среда тестирования

- Операционная система: Windows 10 / Windows 11 (возможны macOS / Linux).
- Java: **JDK 17**.
- UI-фреймворк: **JavaFX**.
- IDE: **IntelliJ IDEA**.
- Дополнительные библиотеки: **Gson** (для JSON).
- Способ запуска: запуск главного класса (App.java или Main.java) как Java Application.



## 4. Виды тестирования

1. **Функциональное тестирование**
   - Проверка основных сценариев:
     - добавление, редактирование, удаление задач;
     - поиск/фильтрация по названию;
     - сохранение и загрузка задач;
     - обновление статистики.

2. **Пользовательское тестирование**
   - Небольшое тестирование с 3–5 одногруппниками.
   - Оценка удобства интерфейса и понятности основных действий.
   



## 5. Сценарии тестирования (основа для тест-кейсов)

1. Создание новой задачи с корректными данными.
2. Попытка добавить задачу с пустым названием (проверка валидации).
3. Редактирование существующей задачи.
4. Удаление задачи с подтверждением.
5. Отмена удаления задачи.
6. Поиск задачи по части названия (совпадение найдено).
7. Поиск задачи по строке, которой нет (совпадений нет).
8. Сохранение списка задач в JSON-файл.
9. Загрузка задач из JSON-файла.
10. Проверка обновления статистики после добавления/удаления задач.
11. Пользовательский сценарий: «новый пользователь добавляет и завершает задачу без подсказок».



## 6. Критерии завершения тестирования

Функциональное тестирование считается успешным, если:

- Все ключевые тест-кейсы (CRUD, поиск, сохранение/загрузка, статистика) имеют статус **Pass** или **Pass с замечаниями**.
- Нет критических ошибок, из-за которых невозможно:
  - запустить приложение;
  - добавить/редактировать/удалить задачи.

Пользовательское тестирование считается успешным, если:

- Большинство участников (не менее 3 из 5) оценивают интерфейс как понятный (4–5 по шкале из 5).
- Все основные действия (добавить, найти, удалить задачу, посмотреть статистику) выполняются с минимальными затруднениями.
